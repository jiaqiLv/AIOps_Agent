## 精化任务

### 1. 异常检测结果（独立章节，必写）

当输入中包含 BLD Metric (ECOD) 异常检测结果时，**必须**在报告中单独输出 `## 异常检测结果` 章节。

该章节应包含：

- **检测方法**：BLD Metric (ECOD)（训练时长、contamination 参数）
- **检测概况**：扫描指标数、检出异常指标数
- **异常指标列表**：按 ECOD score 降序排列，每项含指标名、ECOD 得分、观测值、训练中位数
- **与任务对齐**：任务中声明的异常 KPI 是否在检出列表中
- **检测解读**：异常模式与对根因分析的辅助意义

若检测未执行或失败，仍输出该章节简要说明。

### 2. 根因指标精化

- IAF-RCL + KE-FPC 均命中 → 高置信度，标注 `IAF-RCL+KE-FPC`
- 仅 IAF-RCL → 中置信度；仅 KE-FPC → 低置信度
- 删除与故障无关、或与异常 KPI 无因果路径的指标并说明理由

### 3. 故障类型判断

| 根因指标特征 | 故障类型 |
|---|---|
| cpu_usage / cpu_util | CPU_RESOURCE_SATURATION |
| memory / mem | MEMORY_LEAK |
| latency / rt / duration | LATENCY_SPIKE |
| error_rate / errors | ERROR_SURGE |
| disk / iowait | DISK_BOTTLENECK |
| network / packet | NETWORK_ANOMALY |
| db connection / query | DATABASE_BOTTLENECK |

### 4. 置信度

- 两算法一致且因果清晰：0.8–1.0
- 部分一致：0.6–0.8
- 仅单算法且合理：0.4–0.6
- 结果不明确：0.2–0.4
